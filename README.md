# Система управления складом (WMS)

Веб-приложение для учёта товаров на складе, разработанное на Flask и SQLite.
Поддерживает многопользовательский доступ, учёт движения товаров, аналитику и расчёт себестоимости по методу FIFO.

## Основные возможности

### Справочники
- **Товары** — ведение списка товаров с привязкой к категориям и единицам измерения.
- **Категории** — группировка товаров.
- **Единицы измерения** — создание и управление единицами (шт, кг, л и т.д.).

### Складские операции
- **Приход (IN)** — регистрация поступления товаров с указанием количества и цены закупки.
- **Расход (OUT)** — регистрация отгрузки товаров.
- **Редактирование и удаление** — исправление или отмена ошибочно созданных документов.
- **Произвольная дата операции** — возможность указать дату/время документа вручную.

### Учёт остатков
- **Текущие остатки** — актуальное количество каждого товара.
- **Оценка стоимости** — автоматический расчёт стоимости остатков на основе цен последних поступлений.
- **Фильтрация и поиск** — поиск по названию, фильтрация по категории и статусу (в наличии / нет в наличии).

### История и аналитика
- **История операций** — полный журнал всех операций с пагинацией и фильтрами по дате, типу и товару.
- **Статистика** — сводная таблица движения по всем товарам за выбранный период.
- **Детальная аналитика** — анализ по конкретному товару или категории: объёмы и суммы прихода/расхода, последние операции, график оборота за 12 месяцев.
- **Быстрые периоды** — фильтрация по готовым пресетам: сегодня, неделя, месяц, год.
- **FIFO** — себестоимость расхода рассчитывается по методу FIFO (First-In, First-Out) с точной привязкой к партиям поступлений.

### Пользователи и безопасность
- **Мультипользовательский доступ** — поддержка нескольких учётных записей с ролями `admin` и `user`.
- **Первоначальная настройка** — при первом запуске предлагается создать администратора через веб-интерфейс.
- **Управление пользователями** — администратор может добавлять и удалять пользователей.
- **Смена пароля** — каждый пользователь может изменить собственный пароль.
- **Надёжные пароли** — при создании администратора проверяются требования к сложности пароля (длина, регистр, цифры, спецсимволы).
- **CSRF-защита** — все формы защищены токеном (Flask-WTF).
- **Секретный ключ** — читается из переменной окружения `SECRET_KEY`.

### Персонализация
- **Тёмная и светлая тема** — переключение оформления интерфейса для каждого пользователя отдельно.
- **Часовой пояс** — все даты и время хранятся в UTC и отображаются в настроенном часовом поясе склада.

## Технологии

- **Бэкенд**: Python 3, Flask, Flask-Login, Flask-WTF
- **База данных**: SQLite 3
- **Фронтенд**: HTML, CSS, Alpine.js (без тяжёлых фреймворков)

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone <URL репозитория>
cd <папка проекта>
```

### 2. Виртуальное окружение и зависимости
```bash
python3 -m venv .venv
source .venv/bin/activate       # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Настройка переменных окружения
```bash
cp .env.example .env
```
Откройте `.env` и задайте `SECRET_KEY` — длинную случайную строку.
Сгенерировать можно командой:
```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

### 4. Запуск
```bash
python app.py
```
При первом запуске автоматически создаётся файл `warehouse.db` с нужной структурой таблиц.

### 5. Открыть в браузере
Перейдите по адресу [http://127.0.0.1:5000](http://127.0.0.1:5000) и создайте учётную запись администратора.

## Структура базы данных

| Таблица | Назначение |
|---------|------------|
| `users` | Учётные записи пользователей (логин, хеш пароля, роль, тема) |
| `app_settings` | Настройки приложения (часовой пояс и др.) |
| `categories` | Справочник категорий товаров |
| `units` | Справочник единиц измерения |
| `products` | Товары с текущим остатком |
| `operations` | Заголовки документов прихода/расхода |
| `operation_items` | Строки документов (товар, количество, цена) |
| `lots` | Партии товаров для расчёта FIFO |
| `lot_consumptions` | Привязка расходных строк к конкретным партиям |
