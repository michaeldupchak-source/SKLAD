# SKLAD — Система управления складом (WMS)

Веб-приложение для учёта товаров на складе, разработанное на Flask и SQLite.
Поддерживает многопользовательский доступ, учёт движения товаров, аналитику
и расчёт себестоимости по методу FIFO.

## Возможности

### Справочники
- **Товары** — список товаров с привязкой к категориям и единицам измерения
- **Категории** — группировка товаров
- **Единицы измерения** — шт, кг, л и любые другие

### Складские операции
- **Приход (IN)** — поступление товаров с указанием количества и цены закупки
- **Расход (OUT)** — отгрузка товаров
- **Редактирование и удаление** — исправление ошибочных документов
- **Произвольная дата** — дату и время документа можно указать вручную

### Учёт остатков
- **Текущие остатки** — актуальное количество каждого товара
- **Оценка стоимости** — автоматический расчёт по ценам последних поступлений
- **Фильтрация и поиск** — по названию, категории, наличию

### Аналитика
- **История операций** — полный журнал с пагинацией и фильтрами
- **Статистика** — сводная таблица движения за выбранный период
- **Детальная аналитика** — анализ по товару или категории, график оборота за 12 месяцев
- **FIFO** — себестоимость расхода по методу First-In, First-Out

### Пользователи и безопасность
- **Роли** — `admin` и `user`
- **Первоначальная настройка** — создание администратора через веб-интерфейс при первом запуске
- **CSRF-защита** — все формы защищены токеном (Flask-WTF)
- **SECRET_KEY** — читается из переменной окружения

### Интерфейс
- **Тёмная и светлая тема** — индивидуально для каждого пользователя
- **Часовой пояс** — UTC-хранение, отображение в настроенном часовом поясе
- **PWA** — работает как приложение на мобильных устройствах

## Локальный запуск

```bash
# 1. Клонирование
git clone <URL репозитория>
cd SKLAD

# 2. Виртуальное окружение
python3 -m venv .venv
source .venv/bin/activate       # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# 3. Переменные окружения
cp .env.example .env
# Отредактируй .env: вставь сгенерированный SECRET_KEY
python3 -c "import secrets; print(secrets.token_hex(32))"

# 4. Запуск
python app.py
```

Открой [http://127.0.0.1:5000](http://127.0.0.1:5000) — при первом запуске
предложит создать учётную запись администратора.

## Деплой на сервер

Полная документация: [DEPLOY.md](DEPLOY.md)

### Быстрая установка (одна команда)

Для чистого Ubuntu-сервера достаточно скачать и запустить скрипт установки. Скрипт интерактивно запросит нужные параметры (домен, URL репозитория, порты):

```bash
wget https://raw.githubusercontent.com/YOUR_USERNAME/SKLAD/main/setup.sh
sudo bash setup.sh
```

С SSL-сертификатом (Let's Encrypt) — требует чтобы домен смотрел на IP сервера и Nginx порт был 80:

```bash
sudo bash setup.sh --ssl
```

Скрипт автоматически:
- Спросит домен, репозиторий и порты (Nginx и Gunicorn)
- Установит зависимости (python3, nginx, certbot)
- Создаст системного пользователя `deploy`
- Склонирует репозиторий и создаст виртуальное окружение
- Сгенерирует `SECRET_KEY` и создаст `.env`
- Настроит systemd-сервис и nginx
- Опционально получит SSL-сертификат

### Параметры setup.sh (можно передавать флагами)

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `--domain` | интерактивно | Домен или IP |
| `--repo` | интерактивно | Git URL репозитория |
| `--user` | `deploy` | Системный пользователь |
| `--dir` | `/var/www/sklad` | Директория приложения |
| `--branch` | `main` | Git-ветка |
| `--ssl` | выкл. | Получить SSL через certbot |

## CI/CD

После первичной установки каждый `git push main` автоматически деплоит приложение:

```
git push main
    └─> GitHub Actions
            └─> SSH на VPS
                    ├─> git pull origin main
                    ├─> pip install -r requirements.txt
                    └─> sudo systemctl restart sklad
```

Настройка занимает ~2 минуты. Подробно: [DEPLOY.md → Шаг 4](DEPLOY.md#шаг-4--github-actions-cicd).

## Структура базы данных

| Таблица | Назначение |
|---------|------------|
| `users` | Учётные записи (логин, хеш пароля, роль, тема) |
| `app_settings` | Настройки приложения (часовой пояс и др.) |
| `categories` | Справочник категорий товаров |
| `units` | Справочник единиц измерения |
| `products` | Товары с текущим остатком |
| `operations` | Заголовки документов прихода/расхода |
| `operation_items` | Строки документов (товар, количество, цена) |
| `lots` | Партии товаров для расчёта FIFO |
| `lot_consumptions` | Привязка расходных строк к партиям |
